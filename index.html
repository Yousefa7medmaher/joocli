Installation

Running in 30 seconds

One Python file. No virtual environments. No package managers. Pick your platform.

1
Download joo_cli_v5.py
wget https://raw.githubusercontent.com/Yousefa7medmaher/joocli/master/joo_cli_v5.py
2
Fix CRLF line endings (WSL only)
sed -i 's/\r//' joo_cli_v5.py
Only needed if downloaded on Windows first.
3
Make global and launch
chmod +x joo_cli_v5.py
sudo cp joo_cli_v5.py /usr/local/bin/joo

Extensibility

Build your own commands

The plugin API is intentionally minimal. A plugin is just a Python file with two things: a command list and a handler function.

example_plugin_sysinfo.py → drop in ~/.joocli/plugins/
# JooCLI Plugin — adds :sysinfo command COMMANDS = ["sysinfo"] # commands this plugin registers DESCRIPTION = "Enhanced system reporter" # shown in :plugins list def handle(args: str) -> str: """Called when user types ':sysinfo [args]'""" import subprocess sub = args.strip().lower() if sub == "cpu": return subprocess.getoutput("lscpu | grep 'Model name'") elif sub == "mem": return subprocess.getoutput("free -h") return subprocess.getoutput("uname -a")
✓ Plugin conventions
• Define COMMANDS = ["name"] — list of commands your plugin registers
• Define handle(args: str) -> str — receives everything after the command
• Return a string to print, or None to handle printing yourself
• Plugins get full Python stdlib access and can import anything installed
⚡ Auto-loaded on startup
• All *.py files in ~/.joocli/plugins/ load at launch
• Load errors are shown as warnings — they don't crash the shell
• Tab-completion works for plugin commands automatically
• List all active plugins with :plugins
