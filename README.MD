## What is JooCLI?

JooCLI is a smart terminal shell that wraps your everyday commands with:

- **Streaming AI** ‚ÄĒ ask Claude, ChatGPT, Groq, or Gemini and watch the answer appear word by word
- **Auto error capture** ‚ÄĒ every failed command logs its error and shows an instant fix
- **50+ built-in command docs** ‚ÄĒ `:help grep` shows all flags with a real example
- **Full network toolkit** ‚ÄĒ ping, traceroute, DNS, WHOIS, ARP, port scanning, TCP checks
- **Docker inspector** ‚ÄĒ containers, images, volumes, logs, exec ‚ÄĒ all from one place
- **Pure Python stdlib** ‚ÄĒ no pip, no virtualenv, no dependencies

## ūüď¶ Install

**Option 1 ‚ÄĒ Run directly**
```bash
# Fix Windows CRLF first if on WSL
sed -i 's/\r//' joo_cli.py

python3 joo_cli.py
```

**Option 2 ‚ÄĒ Install as global `joo` command** *(recommended)*
```bash
chmod +x joo_cli.py
sudo cp joo_cli.py /usr/local/bin/joo
joo
```

**Option 3 ‚ÄĒ One-liner install**
```bash
sudo curl -fsSL https://raw.githubusercontent.com/youruser/joocli/main/joo_cli.py \
  -o /usr/local/bin/joo && sudo chmod +x /usr/local/bin/joo && joo
```

> **Requirements:** Python 3.8+ ¬∑ Linux, macOS, or WSL2 ¬∑ No pip packages needed

## ūü§Ė AI Setup

JooCLI supports four AI providers. Pick any one ‚ÄĒ you only need a single key.

| Provider | Speed | Best for | Get key |
|---|---|---|---|
| **Groq / LLaMA** | ‚ö° Fastest | Quick answers, free tier | [console.groq.com](https://console.groq.com) |
| **Claude** | ūüß† Smartest | Complex debugging | [console.anthropic.com](https://console.anthropic.com) |
| **ChatGPT** | ūüĒß Reliable | General use | [platform.openai.com](https://platform.openai.com) |
| **Gemini** | ūüÜď Free | Budget option | [aistudio.google.com](https://aistudio.google.com) |

**Set a key ‚ÄĒ three ways:**

```bash
# 1. Inside JooCLI (saved to ~/.joocli_config.json)
:ai key groq gsk_xxxxxxxxxxxx
:ai key claude sk-ant-xxxxxxxxxxxx

# 2. Environment variable (add to ~/.bashrc or ~/.zshrc)
export GROQ_API_KEY="gsk_xxxxxxxxxxxx"
export ANTHROPIC_API_KEY="sk-ant-xxxxxxxxxxxx"

# 3. Switch active provider anytime
:ai set groq
:ai set claude
```

**All responses stream in real-time** ‚ÄĒ you see the answer as it's generated, not all at once.

## ‚ö° Error Auto-Detection

Every failed command is automatically analyzed. JooCLI recognizes 24+ patterns:

| Error message | Instant fix shown |
|---|---|
| `command not found` | Check spelling, install package, fix `$PATH` |
| `permission denied` | Use `sudo`, check `ls -la`, fix with `chmod` |
| `address already in use` | `lsof -i :<PORT>` ‚Üí `kill -9 <PID>` |
| `connection refused` | `systemctl status <svc>` ¬∑ `ss -tlnp` |
| `no space left on device` | `df -h` ‚Üí `du -sh * \| sort -hr` |
| `name or service not known` | DNS failure ‚Üí check `/etc/resolv.conf` |
| `SSL certificate` | Check system clock ¬∑ `update-ca-certificates` |
| `authentication failed` | Check SSH key perms `chmod 600 ~/.ssh/*` |
| `host unreachable` | `ping` ¬∑ `traceroute` ¬∑ check firewall |
| `out of memory` | `free -h` ¬∑ kill hogs ¬∑ add swap |
| `python3\r` / CRLF | `sed -i 's/\r//' yourfile.py` |
| _anything else_ | `:ai fix the error above` |

## ‚Ěď FAQ

**Q: Ctrl-C crashes the whole shell**
> Fixed in v4.0. JooCLI now uses `Popen` with signal handling so Ctrl-C only interrupts the running command ‚ÄĒ it never crashes the shell.

**Q: `python3\r: No such file or directory`**
> Windows CRLF line endings in the script. Fix: `sed -i 's/\r//' joo_cli.py`

**Q: AI says "no provider configured"**
> Set a key with `:ai key groq YOUR_KEY` or `export GROQ_API_KEY=...` ‚ÄĒ Groq has a free tier.

**Q: `:trace` says tool not found**
> Install traceroute: `sudo apt install traceroute` or `sudo apt install mtr`

**Q: `:net scan` hangs or is slow**
> nmap scans take time on large subnets. For a quick check use `:net check HOST:PORT` instead.

**Q: Does this work on macOS?**
> Yes. `ping` syntax differs slightly (macOS uses `-c` the same way). `traceroute` and `mtr` need `brew install mtr`.

**Q: Can I use multiple AI keys?**
> Yes ‚ÄĒ save all of them with `:ai key <provider> <key>` and switch between them with `:ai set <provider>`.
