⚠️
WSL / VPN Users: Cloudflare 403 Error
Groq API uses Cloudflare which may block requests from WSL or VPS IPs (error code 1010). JooCLI v3.0 sends a browser-like User-Agent header to avoid this in most cases. If you still see a 403, try: :ai set claude or run from a native Windows terminal, or use a VPN.
Installation

Up and running in 30 seconds

One file. One command. Pick your platform.

1
Download joo_cli.py
wget https://raw.githubusercontent.com/Yousefa7medmaher/joocli/master/joo_cli.py
2
Fix line endings (WSL only)
sed -i 's/\r//' joo_cli.py
Only needed if you downloaded on Windows first.
3
Make it a global command
chmod +x joo_cli.py
sudo cp joo_cli.py /usr/local/bin/joo
joo   # launch!
1
Download via curl
curl -O  https://raw.githubusercontent.com/Yousefa7medmaher/joocli/master/joo_cli.py
2
Install as global command
chmod +x joo_cli.py
sudo cp joo_cli.py /usr/local/bin/joo
3
Launch!
joo
macOS ships Python 3 pre-installed. No extra setup needed.
1
Install WSL2 (recommended)
# In PowerShell as Administrator:
wsl --install
Then open Ubuntu and follow the Linux steps above.
2
Native Python (without WSL)
# Download Python from python.org, then:
python joo_cli.py
⚡ Running natively on Windows avoids Cloudflare blocks that affect WSL IPs.
1
Get an API key (choose one)
• console.anthropic.com — Claude (sk-ant-...)
• platform.openai.com — ChatGPT (sk-...)
• console.groq.com — Groq free tier (gsk_...)
• aistudio.google.com — Gemini (AIza...)
2
Set via environment variable
# Add to ~/.bashrc or ~/.zshrc
export GROQ_API_KEY="gsk_your-key-here"
# or for Claude:
export ANTHROPIC_API_KEY="sk-ant-your-key-here"
3
Or save key inside JooCLI
joo ❯ :ai key groq gsk_your-key-here
  Key for 'groq' saved ✓
Keys saved with :ai key are stored in ~/.joocli_config.json
1
Check provider status
joo ❯ :ai status
Shows all 4 providers, key validity, and which is active.
2
Add keys for each provider
joo ❯ :ai key claude   sk-ant-api123...
joo ❯ :ai key chatgpt  sk-proj-abc...
joo ❯ :ai key groq     gsk_xyz...
joo ❯ :ai key gemini   AIzaSy...
Keys are validated by prefix — wrong provider = rejected with a clear message.
3
Switch providers on the fly
joo ❯ :ai set claude
  Switched to Claude (Anthropic) ✓
joo ❯ :ai set groq
  Switched to Groq / LLaMA ✓
Active provider persists across sessions via ~/.joocli_config.json
Features

Everything you need

Built for developers who live in the terminal.

🤖
Multi-Provider AI
Claude, ChatGPT, Groq, and Gemini. Switch providers at runtime. Keys validated by prefix to prevent wrong-provider mistakes.
:ai set <provider> · :ai status
⌨
Tab Autocomplete
Autocompletes command names, flags, and file paths. Knows 35+ commands and every executable in your $PATH.
Tab → complete
🐳
Docker Inspector
Full container report with status, ports, IPs, images, networks, and volumes. Inspect, exec, and manage containers.
:docker · :docker inspect <n>
🌐
Network Tools
See all network interfaces, IP addresses, routing table, DNS servers, and listening ports in one command.
:net · :ports
📖
Command Docs
Built-in reference for 35+ commands: description, every flag explained, and a real-world example.
:help <command>
🔧
Instant Error Fixes
Recognizes 15+ common error patterns and shows a fix immediately. Auto-triggered after any failed command.
:fix · :last
🔑
Key Validation
Each provider has a known key prefix. JooCLI detects if you put a Groq key on Claude (or any mismatch) and warns you clearly.
:ai key <provider> <key>
🛡
Cloudflare Fix
Browser-like User-Agent header on all API calls prevents Cloudflare 403/1010 blocks common on WSL and VPS environments.
Automatic · no config needed
